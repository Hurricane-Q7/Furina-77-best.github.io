1.31-C语言-函数递归用迭代

2026-01-31 3 min read

函数递归

程序调用自身的编程技巧称为递归。
一个过程或函数在其定义或说明中有直接或间接调用自身的
一种方法，它通常把一个大型复杂的问题层层转化为一个与原问题相似的规模较小的问题来求解，
递归的主要思考方式在于:把大事化小，小事化了
递归递归，先递后归！
递与归是两个动作！

void print(unsigned int n)
{
	if (n > 9)
	{
		print(n / 10);
	}
	printf("%d ", n % 10);
}
int main()
{
	unsigned int num = 0;
	scanf("%u", &num);
	print(num);//函数调用完会回到刚开始调用的地方，执行剩下的部分，与返回值什么是void型无关的。
	return 0;
}
x % 10 是编程中提取整数 x 个位数字的经典写法（比如从 123 中提取 3）
余数等于 “被除数” 减去 “除数乘以商” 的结果

递归的两个必要条件

1.存在限制条件，当满足这个限制条件的时候，递归便不再继续。

  1. 每次递归调用之后越来越接近这个限制条件。
    死递归：函数一直重复调用自己，没有中止条件，最终会导致在内存栈区造成栈溢出。
    递归次数太多容易出现栈溢出(Stack overflow)的情况！
    那如何解决上述的问题:
    1.将递归改写成非递归。
    2.使用static对象替代nonstatic局部对象。在递归函数设计中，可以使用static对象替代nonstatic局部对象(即栈对象)，这不
    仅可以减少每次递归调用和返回时产生和释放nonstatic对象的开销，而且static对象还可以保存递归调用的中间状态，并且可为各个调用层所访问。
    函数递归的几个经典项目:
  2. 汉诺塔问题
  3. 青蛙跳台阶

迭代

循环是一种迭代！

int fib(int n)
{
	int count = 1;
	int a = 1;
	int b = 1;
	int c = a + b;
	int i = 3;
	if (n <= 2)
	{
		return 1;
	}
	else//n>2的情况,当n等于三的时候，c=a+b执行一次，n=4的时候a+b执行两次
	{
		while (count<=n-2)
		{
			
			c = a + b;
			a = b;
			b = c;
			count++;
		}
	}
	return c;
}
int main()
{
	int n = 0;
	printf("请输入一个数字，以求斐波那契数：");
	scanf("%d", &n);
	int ret = fib(n);
	printf("该数字对应的斐波那契数字是：%d\n", ret);
	return 0;
}
