C语言实现windows的dir cd mkdir指令

2026-01-21 4 min read

下面是我感觉值得记载的东西!

<direct.h>头文件

核心含义
#include <direct.h>
是Windows 系统专属的 C 语言头文件（不属于 ANSI C 标准），它声明了一系列用于操作文件目录 / 文件夹的函数，比如创建目录、获取当前工作目录、更改工作目录等。
简单来说：这个头文件是 Windows 平台给 C 语言程序提供的目录操作工具箱没有包含它，就无法调用这些目录相关的函数。

<direct.h>包含的三个函数

_getcwd函数

Parameter
buffer：一个字符数组（缓冲区），用来存放当前工作目录的路径字符串!
maxlen:缓冲区的最大长度（需要包含字符串结束符\0)
函数原型
char * _getcwd ( char * buffer,int maxlen);
Return Value

  • 成功时返回指向buffer的指针；
  • 失败时返回NULL（空指针）

_chdir函数

函数功能：改变当前程序的工作目录。
函数原型：int _chdir(const char * dirname)
参数：dirname： const char *类型，代表要切换到的目标目录路径。
相对路径：如 "../"（上级目录）、"sub_dir"（当前目录下的 sub_dir 子目录）；
绝对路径：如 "D:\test"（Windows）、"/home/user"（Linux）。
注意：Windows 下路径分隔符用 \（转义后）或 /，不能直接用 \。
Return Value：

  1. 成功：返回0；
  2. 失败：返回-1；

_mkdir函数

函数原型：
int _mkdir(const char *dirname);
函数功能：
在指定路径下创建一个新的空目录（文件夹）
参数：
dirname：const char * 类型，代表要创建的目录路径，支持两种格式：

  • 相对路径：如 "new_dir"（在当前工作目录下创建名为 new_dir 的文件夹）、"sub_dir/test"（需确保 sub_dir 已存在，否则创建失败）；
  • 绝对路径：如 "D:\my_files\temp"（Windows）、"/home/user/new_dir"（Linux）。
    关键注意：Windows 下路径分隔符用 \（转义后）或 /，不能直接用 \；且 _mkdir 只能创建最后一级目录，如果上级目录不存在（比如想创建 a/b/c，但 a/b 不存在），会创建失败。
    返回值：
    成功：返回 0；
    失败：返回 -1，同时会设置全局变量 errno 标识错误原因（需包含 errno.h 头文件）。

<io.h>头文件的三个函数

该代码中<io.h>header file主要调用_findfirst(初始化查找) _findnext（循环查找） _findclose（释放资源）函数，这些函数是Windows专用的遍历目录函数。

struct _finddata_t file_info;

file_info 是 _finddata_t 类型的结构体变量（定义在io.h中）,用于存储文件/文件夹的属性信息，核心成员有：

_findfirst

实现功能：
查找指定目录下第一个匹配指定模式的文件 / 文件夹，获取其句柄和属性信息。
函数原型：
long _findfirst(const char *filename, struct _finddata_t *fileinfo);
参数：

  1. filename：匹配模式（如 "*" 匹配所有文件 / 文件夹）；
  2. fileinfo：指向 _finddata_t 结构体的指针，用于存储找到的文件信息。
    返回值：
    成功返回一个有效句柄（非-1的long int型）
    失败返回-1（目录为空，权限不足）

_findnext

功能：基于 _findfirst 获取的句柄，查找下一个匹配的文件/文件夹。
函数原型
int _findnext(long handle, struct _finddata_t *fileinfo);
参数：
handle：_findfirst 返回的有效句柄；
fileinfo：同上，存储下一个文件的信息。
返回值：
成功：返回 0；
失败：返回 -1（如已遍历完所有文件）。

_findclose

实现功能：功能：关闭 _findfirst 获取的句柄，释放系统资源（必须调用，否则会内存泄漏）
函数原型：
int _findclose(long handle);
参数：handle：_findfirst 返回的句柄
返回值：
成功：返回0；
失败：返回-1（如句柄无效）；
