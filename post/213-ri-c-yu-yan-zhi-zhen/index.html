指针

首先理解指针的两个要点

内存中有很多个小的内存单元，每个内存单元有1Byte。

  1. 指针是内存中一个最小单元的编号，也就是地址。
  2. 平时口语中说的指针，通常指的是指针变量，用来存放内存地址的变量。

这里我们就明白:

  1. 在32位的机器上，地址是32个0或者1组成二进制序列，那地址就得用4byte的空间来存储，所以一个指针变量的大小就应该是4byte。
  2. 那如果在64位机器上，如果有64个地址线，那一个指针变量的大小是8byte，才能存放一个地址，所以在64位机器上一个指针变量大小就是8byte。

指针和指针类型

指针类型的两条意义

结论1： 指针类型决定了指针在被解引用的时候访问几个字节，有多大的权限（能操作几个字节）
比如，int*的指针变量，被解引用时可以访问4个字节,char *访问1个字节。

int main()
{//x86
    int a = 0x11223344;
    char* pc = (char*)&a;
    *pc = 0;//经过调试，发现仅仅更改了1byte的数据：0x00CFFC50  00 33 22 11（0x表示这是16进制的表现形式,00 33 22 11 ，一共8位，两位一字节）
    return 0;
}
int main()
{
        int a = 0x11223344;
        int* pc= &a;
        *pc = 0;//此时a在内存中是0x 00 00 00 00，一次性被操作了4byte
        return 0;
    
}

结论2： 指针类型决定了指针+-1操作的时候，跳过或后退几个字节，决定了步长！

int main()
{
	int n = 10;
	int* pa = &n;
	char* pc = (char* ) & n;
	printf("%p\n", pa);
	printf("%p\n", pa + 1);//跳过1个int型，也就是4byte，也可以理解位跳过4个内存单元，每个内存单元对应一个地址
	printf("%p\n", pc);
	printf("%p\n", pc+1);//跳过1个char型，也就是1byte
	return 0;
}

注意：即使两个指针变量解引用操作时都可以访问相同的字节、+-时改变的步长一样也不能通用，因为二者指向的对象在内存中的存储方式不一样的！
